The goal shifted from “2 TiB” to “the memory to run the model at 1M full quality, as cheaply as an official-brand build allows.” Answer: it fits far under 2 TiB — and the whole thing turns on what “full quality” actually means for this model family.

Verdict

~₪25–28k landed — won, narrowly, at native precision + MSRP import.

Running Kimi at its native INT4-QAT precision (which is full quality — FP16 is a phantom that adds VRAM, not quality) with the KV cache un-quantized (FP16), the MTP module, and the full vision stack all included, a K2-scale model @ 1M fits in ~700 GiB. That is 45 × 16 GiB DDR5-5600 UDIMM (Crucial CT16G56C46U5) = 720 GiB @ 1.79 TB/s = $6,525 ≈ ₪20k, ~₪25–28k landed in Israel with 18% VAT. Under the ₪30k line — with three honest asterisks: it needs native INT4 (not FP8), Crucial-US MSRP sourcing (Israeli retail is ~2× → ₪40–46k), and it is DRAM-only (the converter silicon is separate).

02The precision reality — why FP16 is a phantom

Kimi’s native, shipped, benchmarked precision is INT4-QAT (K2.5/2.7 model cards: “Native INT4 Quantization,” lossless-claimed) or FP8 (K2-Instruct). The weights are INT4/FP8; up-casting to FP16 decompresses them for zero quality gain at 2–4× the VRAM. So “run it the way you’re supposed to, not over-quantized” = the native checkpoint — not FP16. Where un-quantized precision genuinely matters is exactly where it was required: the KV cache → FP16. Both honored.

03The config & bandwidth

45 × Crucial DDR5-5600 UDIMM (official Micron brand, JEDEC-native CL46, non-ECC), density chosen by precision target — 16 GiB (CT16G56C46U5) for native-INT4, up to 64 GiB (CT64G56C46U5) for a 2× K3.

  • Bandwidth: 45 ch × 44.8 GB/s = 2,016 GB/s raw → the front’s 1,792 GB/s (512-pin × 28 Gb/s) at 88.9% duty — the 11.1 pp gap is the refresh/turnaround headroom.
  • Why 5600 not 6400: 5600 is JEDEC-native (no XMP/EXPO overclock dependency — ideal for a custom controller); official-brand value 6400 UDIMMs are scarce as singles.

05Physical — a hierarchical appliance, never one package

  • Pins: ~110–120 signals/channel → ~5,000–5,400 signal pins for 45 channels (~15k+ balls monolithic). Beachfront ~112–158 mm die-edge vs ~118 mm reticle perimeter → impossible on one die.
  • Required structure: ~9 leaf converters × 5 channels (~2,000-ball FCBGA each) + a switch-class root (~14 Tb/s SerDes/side). 2.5× the pins and ~3× the leaf dies of the 18-MRDIMM design.
  • Board / power: 288-pin UDIMM 133×31 mm, ~4 banks × 12 slots ≈ ~0.2 m² baseboard; ~440–500 W (DRAM + converters) — same class as the 18-MRDIMM build, for half the capacity but ~5–10× lower $/GiB.

07Honest flags (what stays unverified)

  • INT4-vs-FP8 is the hat’s hinge. The win requires accepting native INT4-QAT as full quality (it is, per Moonshot’s lossless claim + INT4 benchmarks). Demand FP8 and it’s ₪38–44k; FP16 is a phantom.
  • Prices: MSRP from Wayback (crucial.com live blocked); most US retail bot-walled; FX approximate; crunch prices move weekly.
  • Timings: tRFC2/tRFCsb numerics are commonly-published JESD79-5 values (JEDEC/Micron die datasheets login-walled); Kingston’s 48 GB sheet prints a 295 ns tRFC that looks like 16 Gb boilerplate — the worse 410 ns was used. The 3DS ~84% figure is spec-derived.
  • Scope: DRAM only. The converter silicon (custom N3E ASIC ~$50M NRE, or a $12–25k FPGA prototype) is separate and far outside ₪30k.
  • Model: no shipped Kimi does 1M yet (256K today); 1M is the forward requirement, correctly budgeted. K3 is not public — K3=1.5–2× is a stated scale-up assumption.

Bottom line. At the model’s genuine full-quality native precision (INT4-QAT), with the KV cache un-quantized, MTP resident, and the full vision stack — all measured, not assumed — a 1M-context Kimi-K2-scale machine needs ~700 GiB, which is 45 × 16 GiB DDR5-5600 UDIMM at 1.79 TB/s for ~$6,525 ≈ ₪25–28k landed: under the ₪30k line. It passes both correctness checks (per-bank refresh mandatory; add converter CRC for the non-ECC UDIMMs) and is buildable as a ~9-leaf hierarchical appliance (~0.2 m², ~0.5 kW), never one package. The hat is off — provided you run native INT4, buy at MSRP, and count the memory alone. FP8, Israeli-retail sourcing, or a 2× multimodal K3 each push it back over.
